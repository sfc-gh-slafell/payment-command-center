01 — Pipeline Overview

End-to-End Data Flow

Data moves through five hops, two ingest paths, and branches into parallel hot + analytical serving layers.

⚡

Event Generator

generator/producer.py · confluent_kafka
Synthetic payment auth events at configurable rate (default 500/sec, up to 50k via docker-compose). Pre-generated pool of 50k events, dispatched at 10 Hz. JSON messages keyed by merchant_id with zstd compression.
< 1ms
📦

Kafka Topic: payments.auth

Apache Kafka 4.0 (KRaft) · 48 partitions · 72hr topic retention (broker: 1hr / 200 MB)
Single topic: payments.auth. Partition key = merchant_id ensures per-merchant ordering.
⚡

HP Connector v4.x

Snowpipe Streaming · Primary Path
Real-time streaming sink with user-defined pipe. Extracts RECORD_METADATA for Kafka lineage.
5-10s latency
📊

Python Fallback Relay

fallback_ingest/relay.py · consumer group: snowflake-fallback-relay
Alternative Kafka consumer on the same payments.auth topic. Batches events (1,000 rows / 5s timeout) and writes via write_pandas to AUTH_EVENTS_RAW. Activated when HP Connector is unavailable. Separate consumer group avoids offset conflict.
1-3 min latency
🗃

PAYMENTS_DB.RAW.AUTH_EVENTS_RAW_V4

Snowflake Standard Table · 22 columns · 14-day retention
Landing table for HP Connector v4 ingest (fallback relay writes to AUTH_EVENTS_RAW). Logical isolation via env column (dev / preprod / prod). Kafka metadata preserved.

02
Kafka → Snowflake RAW
5-10s

Snowpipe Streaming via HP Connector

The HP Kafka Connector v4.x operates in user-defined pipe mode. A Terraform-managed pipe controls column mapping and extracts RECORD_METADATA for Kafka lineage tracking.

-- terraform/pipes.tf (COPY INTO mapping) SELECT $1:env::VARCHAR(16), $1:event_ts::TIMESTAMP_NTZ, $1:event_id::VARCHAR(64), -- ... 13 more business columns ... $1:RECORD_METADATA:topic::VARCHAR(128), $1:RECORD_METADATA:partition::NUMBER, $1:RECORD_METADATA:offset::NUMBER, CURRENT_TIMESTAMP() FROM TABLE(DATA_SOURCE(TYPE => 'STREAMING'))

Why Terraform manages the pipe: CREATE OR REPLACE PIPE invalidates the active streaming channel. Terraform's IF NOT EXISTS is idempotent and safe.

03
RAW → SERVE
60s

Interactive Tables — Hot Serving Path

Two interactive tables materialize from AUTH_EVENTS_RAW_V4 with 60-second TARGET_LAG. PAYMENTS_REFRESH_WH (standard) materializes; PAYMENTS_INTERACTIVE_WH (interactive) serves queries.

IT_AUTH_MINUTE_METRICS (22 columns)

Minute-level aggregates. 2-hour rolling window. No deduplication needed — HP v4 Snowpipe Streaming guarantees exactly-once delivery per Kafka offset.

event_minute env merchant_id region card_brand issuer_bin event_count decline_count approval_count error_count latency_sum_ms 6 histogram buckets total_amount

Latency stored as SUM/COUNT for correct weighted reaggregation across dimension slices.

IT_AUTH_EVENT_SEARCH (16 columns)

Event-level records for drill-down. 60-minute rolling window, ordered by event_ts DESC.

event_ts event_id payment_id merchant_id auth_status decline_code auth_latency_ms + 9 dimension cols

04 — Access Control

Roles & Warehouses

Each hop operates under a specific role with least-privilege access to the resources it needs.

🔐 Ingest Role

PAYMENTS_INGEST_ROLE

HP Connector + fallback relay. INSERT on RAW table, OPERATE on pipe. No access to SERVE or CURATED.

💻 App Role

PAYMENTS_APP_ROLE

Dashboard backend (SPCS). SELECT on SERVE + RAW. BIND SERVICE ENDPOINT. No write access.

👁 Ops Role

PAYMENTS_OPS_ROLE

Dashboard viewers. USAGE on SERVE schema only. Read-only operational access.

⚙ Admin Role

PAYMENTS_ADMIN_ROLE

Terraform, schemachange, dbt. ALL PRIVILEGES on PAYMENTS_DB. Parent of all other roles.

Notable Patterns

Pipe Ownership Split

terraform/pipes.tf + schemachange V1.6.0+

Terraform uses CREATE PIPE IF NOT EXISTS to avoid destroying the active Snowpipe Streaming channel. schemachange uses CREATE OR REPLACE for definition updates, re-granting MONITOR + OPERATE after each replacement.

Import Blocks

terraform/imports.tf · 13 resources

Permanent import blocks (v2.x compound IDs like "PAYMENTS_DB|RAW") enable idempotent terraform apply on cold deploys where state is missing but objects exist.

Privilege Re-Granting

schemachange V1.13.0, V1.14.0

CREATE OR REPLACE resets all grants on the object. Migrations explicitly re-grant MONITOR, OPERATE, and SELECT after every pipe or table recreation.

IT Source Migration

schemachange V1.15.0

Interactive tables switched from AUTH_EVENTS_RAW to AUTH_EVENTS_RAW_V4. Removed QUALIFY ROW_NUMBER() deduplication — HP v4 Snowpipe Streaming guarantees exactly-once delivery per Kafka offset.

06 — Technology Stack

Technologies & Their Purpose

Every tool in the pipeline and why it was chosen.

Apache Kafka (KRaft)

Event Streaming

Decouples generator from Snowflake. 72h topic retention provides replay window for recovery (broker-level override may apply in demo environments). KRaft mode eliminates Zookeeper dependency. zstd compression minimizes network cost.

Snowpipe Streaming (HP Connector)

Real-Time Ingest

5-10 second latency from Kafka to queryable rows. User-defined pipe mode enables typed column mapping and RECORD_METADATA extraction for lineage.

Snowflake Interactive Tables

Sub-Minute Serving

Pre-aggregated metrics with 60-second refresh. Interactive warehouse type optimized for selective queries. Avoids dynamic table stacking latency.

dbt + Dynamic Tables

Analytical Layer

Parallel curated path for BI/ML. Enrichment, hourly/daily aggregations. NOT in the hot serving path — decoupled from dashboard latency.

FastAPI + Dual Pools

API Backend

Dual warehouse routing: interactive WH for SERVE queries, standard WH for RAW freshness. SPCS token auth in production, key-pair for local dev.

React + SPCS

Dashboard UI

Single container serves both static React assets and Python API. Deployed to Snowpark Container Services with OAuth authentication.

Terraform

Infrastructure as Code

Manages stable infrastructure: roles, warehouses, compute pools, pipes, stages. Import blocks for idempotent apply on existing state.

schemachange

Schema Migrations

Versioned DDL for tables, interactive tables, and grants. 15 migrations (V1.0.0–V1.15.0) from raw landing table through complete privilege configuration.
